from pathlib import Path
import pandas as pd

def count_files(folder, exts):
    folder = Path(folder)
    if not folder.exists():
        return 0
    return sum(1 for f in folder.rglob("*") if f.is_file() and f.suffix.lower() in exts)

def count_dataset(base_path):
    base = Path(base_path)

    return {
        "train_images": count_files(base / "train" / "images", {".jpg", ".jpeg", ".png"}),
        "valid_images": count_files(base / "valid" / "images", {".jpg", ".jpeg", ".png"}),
        "test_images": count_files(base / "test" / "images", {".jpg", ".jpeg", ".png"}),

        "train_labels": count_files(base / "train" / "labels", {".txt"}),
        "valid_labels": count_files(base / "valid" / "labels", {".txt"}),
        "test_labels": count_files(base / "test" / "labels", {".txt"}),
    }

summary = []

for name, path in downloaded_paths.items():
    stats = count_dataset(path)
    stats["dataset"] = name
    summary.append(stats)

df_summary = pd.DataFrame(summary)

Dataset Characteristics¶

The datasets show significant differences in size and structure.

  • The crack and house datasets are substantially larger than the others
  • The paint dataset is very small and only contains test samples
  • The mold dataset only contains training data
  • The house dataset lacks a validation split

This indicates strong heterogeneity across datasets.

Implications¶

These differences have several implications:

  • Larger datasets (crack, house) will dominate model learning
  • Missing splits require careful handling during training and evaluation
  • Small datasets (paint) may not contribute significantly to training
  • The model must generalize across datasets with inconsistent structures

This variability reflects realistic conditions and is relevant for evaluating robustness.

def boxes_per_image(path):
    counts = []
    path = Path(path)

    for file in path.rglob("*.txt"):
        with open(file) as f:
            counts.append(len(f.readlines()))
    return counts

for name, path in downloaded_paths.items():
    label_path = Path(path) / "train" / "labels"

    if not label_path.exists():
        continue

    counts = boxes_per_image(label_path)

    if len(counts) == 0:
        continue

    plt.figure()
    plt.hist(counts, bins=20)
    plt.title(f"Boxes per Image - {name}")

Relevance to the Research Objective¶

The identified dataset characteristics directly impact the comparison between YOLOv8 and LLaVA.

YOLOv8 relies on structured bounding box annotations and is therefore sensitive to:

  • dataset imbalance
  • annotation consistency
  • availability of training and validation splits

In contrast, LLaVA operates as a vision-language model and may be less dependent on strict annotation structures, potentially allowing it to generalize better across heterogeneous datasets.

This difference makes the dataset variability particularly relevant for evaluating the robustness of both approaches.

Implications for Methodology¶

Based on the EDA, the following steps are required:

  • Standardization of labels across datasets
  • Handling of missing dataset splits
  • Careful interpretation of results due to dataset imbalance

These steps ensure a fair and meaningful comparison between YOLOv8 and LLaVA.
